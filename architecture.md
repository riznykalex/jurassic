## 1. Огляд проєкту

Проєкт є веб-симуляцією (RTS-style), де юніти (динозаври) взаємодіють у реальному часі на ігровому полі. Архітектура побудована за принципом розділення логіки симуляції (Physics/State) та відображення (DOM Rendering).

---

## 2. Діаграма архітектури (Потоки даних)

```text
[Input Module] ---> [Event Bus] ---> [Simulation Engine]
                                            |
                                            v
[Renderer] <--------------------------- [State Store]
    |
    +---> [DOM Elements] (Units, Food, UI)
    +---> [Styles/CSS Animations]

```

---

## 3. Опис компонентів

### Core (Ядро)

* **`simulation.js`** (`SimulationEngine`):
* Є «мозком» проєкту.
* Зберігає стан усіх об'єктів: `balls` (юніти), `foods`, `poisons`.
* Відповідає за логіку руху (`vx`, `vy`), атаки, вибір цілей та розрахунок життєвих показників.
* Не має знань про DOM або стилі.

### Presentation (Відображення)

* **`renderer.js`** (`Renderer`):
* Відповідає за перетворення даних з `SimulationEngine` у візуальні елементи на сторінці.
* Керує життєвим циклом DOM-елементів (створення/видалення через `Map` кеш).
* Виконує синхронізацію: зчитує стан юніта -> застосовує CSS-класи -> оновлює `backgroundPosition` для анімації спрайтів.

### Utilities & Config

* **`utils.js`**:
* Набір допоміжних функцій.
* Містить критично важливу функцію `spriteFramePosition`, яка обчислює координати кадру спрайту на основі часу (`now`), стану юніта та напрямку.

* **`config.js`**:
* Централізоване сховище констант: `SPECIES` (види динозаврів), `BALANCE` (швидкість, радіуси), `WIDTH`/`HEIGHT` ігрового поля.

### Input & Events

* **`input.js`**:
* Обробка взаємодії користувача (кліки, перетягування рамки, скрол карти правою кнопкою миші).
* Транслює дії миші у відповідні команди для симуляції.

* **`events.js`**:
* Шина подій (Event Bus) для слабкого зв'язку між компонентами.

## 4. Життєвий цикл рендерингу (The Render Loop)

Рендеринг відбувається через ітерацію по колекції `balls` у `SimulationEngine`:

1. **Синхронізація (Sync):** Для кожного юніта перевіряється, чи існує DOM-елемент. Якщо ні — створюється.
2. **Обчислення стану:** Визначається `rawState` (`idle`, `walk`, `attack`) на основі фізичних прапорців та швидкості.
3. **Логіка повороту:** Розраховується напрямок руху (`dir-up`, `dir-down`, тощо) та стан `facingLeft`.
4. **Анімація:** Викликається `spriteFramePosition`, яка повертає рядок для `background-position`.
5. **Оновлення DOM:** Застосовуються зміни (позиція `left`/`top`, `className`, `transform: scale`).
---

## 5. Взаємодія з CSS

* Проєкт використовує **Sprite Sheets**.
* CSS-класи (наприклад, `.state-walk`) використовуються для загальної стилізації.
* JS-інлайн стилі (`style.backgroundPosition`) використовуються для покадрової анімації, оскільки вона залежить від часу (`performance.now()`).
* **Важливо:** Анімації через `@keyframes` у CSS для `.ball.has-sprite` вимкнені через `animation: none !important`, щоб уникнути конфліктів із JS-керуванням кадрами.

---

## 6. Як додати нову функціональність?

1. **Якщо потрібно додати новий тип динозавра:** Додайте його в `SPECIES` у `config.js`.
2. **Якщо потрібно змінити поведінку (фізику):** Змінюйте логіку в `simulation.js`.
3. **Якщо потрібно змінити візуальну частину:** Редагуйте `renderer.js` для логіки оновлення DOM або `utils.js` для зміни математики анімацій.
